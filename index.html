自己 PR

インフラエンジニアとして

配属されて半年でサービスの構築 / 運用の主担当を務め、「構築 / 運用の効率化」と「迅速な障害復旧」を実現しました。前者では、手順書から設定値と手順を分離し、スクリプト化することで工数を約 1/3 に削減しました。また、クラウドの新機能を ウォッチし、Azure Automation や Logic Apps による部分的な作業の自動化、Log Analitycs によるログ集約など、クラウドの機能を積極的に活用して運用負担とコストを削減しました。後者では、Twilio による電話通報の導入と障害切り分け手順の見直しにより、障害発生から復旧までの時間を平均で 2/3 に削減しました。ただ、上手くいったことばかりではなく、チームに新しい手法や文化を根付かせることの難しさ、証明書関連で 2 度の障害を発生させてしまうなどサービスの安定稼働を実現する難しさを経験することができました。

海外拠点と連携したサービス提供

問い合わせ対応 / 定例会議 / ノウハウ共有など、ほぼ毎日海外拠点のメンバーと接してきました。意思決定が早い代わりにリスク管理が甘いこと、管理職と社員の間に距離があり情報が抜け落ちるなどの課題がありました。前者では、リスクを事前に提示するだけでなく、なぜそういうリスクが起こりうるかの背景を説明し、 組織文化としてリスク管理が根付くよう働きかけました。後者では、管理職と社員の間のパイプラインとして機能することで情報を双方に共有し、認識のズレを埋めました。お互いの良さは活かしつつ、悪い部分を補うことで、強力なチームを作り上げることができることを学びました。 各国のメンバーと関われたことは、私にとって非常に貴重な経験でした。困難に立ち向かう上でチームを団結させて士気を高める方法や、良い働きをしたメンバーを全体の場で積極的に褒める文化の良さを学ぶことができました。
